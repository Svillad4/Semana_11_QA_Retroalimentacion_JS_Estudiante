# Plan de mejora

El plan de mejora permite transformar los hallazgos de QA en acciones concretas para fortalecer el proyecto.

## Objetivo del plan

Describe qué se busca mejorar en la siguiente versión del proyecto.

**Objetivo:**
En futuras versiones del proyecto se podrían implementar mejoras como un sistema de búsqueda de registros, filtros dinámicos, exportación de datos y estadísticas visuales. También sería útil agregar mayor seguridad en validaciones, un diseño más responsive para celulares y nuevas funciones automáticas que mejoren la experiencia del usuario y la organización de la información

## Reflexión final
Durante esta actividad comprendí la importancia de revisar, probar y corregir constantemente un proyecto antes de entregarlo. Gracias a la depuración y a la retroalimentación recibida fue posible identificar errores que afectaban el funcionamiento del sistema y mejorarlos de forma organizada. También aprendí que una buena integración entre HTML, CSS y JavaScript es fundamental para crear aplicaciones más estables, funcionales y fáciles de usar 
Responde:

1. ¿Cuál fue el error más importante encontrado?
2. ¿Qué aprendiste sobre depuración sistemática?
3. ¿Cómo ayudó la retroalimentación de pares a mejorar el proyecto?
4. ¿Qué harías diferente en una próxima entrega?

**Respuesta:**
1. El error más importante encontrado fue la diferencia entre los IDs del HTML y JavaScript, porque impedía que el formulario, los botones y la tabla funcionaran correctamente dentro del sistema.

2. Aprendí que la depuración sistemática ayuda a encontrar errores de manera más organizada usando la consola del navegador, revisando validaciones y probando paso a paso cada función del proyecto.

3. La retroalimentación de pares ayudó a detectar errores que no había notado antes, especialmente problemas en validaciones, lógica y funcionamiento de algunos botones y cálculos.

4. En una próxima entrega organizaría mejor las validaciones desde el inicio, revisaría más veces la consola y haría pruebas más completas antes de finalizar el proyecto para evitar errores grandes
